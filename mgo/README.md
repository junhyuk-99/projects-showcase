# MGO – Equipment Operation Monitoring (MON)

## Context
현장의 **SCADA 설비 데이터**는 존재했지만,  
설비 가동 여부와 상태를 **Web에서 한눈에 파악하기 어려운 구조**였습니다.

설비별 가동/비가동 상태, 주요 지표를  
현장·관리자가 동일하게 확인할 수 있는 **설비 가동 모니터링 화면**이 필요했습니다.

---

## My Role
- **설비 가동 상태 중심** 모니터링 화면 설계
- SCADA 설비 데이터 흐름 정리 및 **API 연동 구조 설계**
- 설비 상태 판단 기준을 Web 화면에 표현하는 로직 정의

---

## Key Design

### 1. 설비 가동 상태 중심 모델링
- 설비를 **가동 / 비가동 / 대기** 상태로 단순화하여 표현
- 현장 기준 신호를 상태 값으로 변환하는 규칙 정의

### 2. 실시간 모니터링 화면 구성
- 설비별 상태를 한 화면에서 확인 가능한 구조 설계
- 상태 변화 시 즉시 인지할 수 있도록 시각적 구분 적용

### 3. 현장–관리자 공통 시점 제공
- 현장 설비 상태를 Web에서 실시간 공유
- 관리자·운영자 간 설비 정보 해석 차이 최소화

---

## Result (Example)
- 설비 가동 상태 확인 및 이상 판단 흐름 단순화
- 현장 의존적 확인 절차를 Web 기반 모니터링으로 전환
- 설비 정보 공유 및 대응 속도 개선
