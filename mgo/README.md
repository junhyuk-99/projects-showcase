# MGO – Equipment Operation Monitoring (MON)
SCADA 설비 데이터 기반 실시간 가동 모니터링 시스템

## 배경

현장에는 SCADA를 통해 설비 데이터가 수집되고 있었지만,
설비 가동 여부나 상태를 **Web에서 한눈에 파악하기 어려운 구조**였습니다.

현장 담당자와 관리자가 **같은 기준으로 설비 상태를 확인**할 수 있는
모니터링 화면이 필요했습니다.

---

## 담당 업무
- 설비 가동 상태 중심의 **모니터링 화면 설계**
- SCADA 설비 데이터 흐름 분석 및 **Web API 연동 구조 설계**
- 현장 신호 → 상태 값 변환을 위한 **설비 상태 판단 기준 정의**
- Spring Boot 기반 API 개발 및 React 화면 구현

---

## 설계 포인트

### 1. 설비 상태 모델링
- 설비 상태를 **가동 / 비가동 / 대기** 3가지로 단순화
- SCADA 신호를 상태 값으로 변환하는 규칙 정의
- 화면에서는 상태 변화에만 집중할 수 있는 구조

### 2. 실시간 모니터링 화면
- 설비별 상태를 한 화면에서 확인 가능하도록 구성
- 상태 변화 시 바로 인지할 수 있도록 시각적 구분 적용
- 폴링 기반으로 주기적 데이터 갱신 처리

### 3. 현장–관리자 동일 시점 제공
- 현장 설비 상태를 Web에서 실시간으로 공유
- 설비 상태에 대한 해석 차이를 줄이기 위해 판단 기준 통일

---

## 결과
- 설비 가동 상태 확인 및 이상 판단 흐름 단순화
- 현장 방문 없이 Web에서 설비 현황 확인 가능
- 현장-관리자 간 설비 정보 공유 및 대응 속도 개선
