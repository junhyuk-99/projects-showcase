# NWM – Real-time Data Tracking & Monitoring
실시간 데이터 자동 집계·정합 및 상태 모니터링 Web 시스템

## 배경

외부 장비에서 수집되는 데이터를 기반으로 두 가지 문제를 해결해야 했습니다.

**1. 데이터 정합 문제**

카운트 데이터를 운영 시스템의 실적과 LOT 단위로 정합시켜야 하는데,
- **연속 투입 데이터**: 카운트는 계속 쌓이지만, LOT 확정이 지연되는 경우가 많음
- **개별 이벤트 데이터**: 이벤트 자체는 명확하지만 카운트·순번이 어긋나는 경우 발생

LOT 입력이 지연되거나 누락되면 **이력에 공백**이 생기는 문제가 있었습니다.

**2. 실시간 모니터링 부재**

데이터는 수집되고 있었지만, 운영 상태를 **Web에서 한눈에 파악하기 어려운 구조**였습니다.
담당자와 관리자가 같은 기준으로 상태를 확인할 수 있는 모니터링 화면이 필요했습니다.

## 담당 업무
- 카운트 기반 **자동 집계 구조 설계**
- **업무일자 기준(06:30)** 집계·정합 로직 정의
- SQL Server Stored Procedure 설계 및 운영 안정화
- Web 시스템 연동을 위한 LOT/카운트 조회 API 구조 정리
- 실시간 상태 모니터링 **화면 설계 및 API 개발**
- 원시 신호 → 상태 값 변환을 위한 **상태 판단 기준 정의**

---

## 설계 포인트

### 1. 시간 스트림 기반 매칭
- 이벤트를 **시간순(ROW_NUMBER)** 으로 정렬
- 카운트 슬롯과 이벤트를 순차 매칭하여 순번 어긋남 방지

### 2. 데이터 유형별 트래킹 분리
- **연속 투입형**
  - 카운트 기준으로 슬롯을 먼저 생성
  - LOT 정보가 늦게 들어와도 이후 자동 보정되는 구조
- **개별 이벤트형**
  - 이벤트 단위 매칭
  - 중복/노이즈 이벤트 제거 후 유효 데이터만 반영

### 3. 외부 시스템 지연 대응
- 외부 데이터 소스 → 운영 시스템 간 연계 지연을 고려한 보완 경로 설계
- 데이터 오류 발생 시 사후 정합으로 수정 가능하도록 구성

### 4. 실시간 상태 모니터링
- 상태를 **가동 / 비가동 / 대기** 3가지로 단순화하여 모델링
- 폴링 기반 주기적 데이터 갱신 처리
- 상태 변화 시 바로 인지할 수 있도록 시각적 구분 적용

## 결과
- 카운트 기반 자동 집계로 **수기 입력·검증 작업 감소**
- LOT 누락/중복으로 인한 **이력 공백 문제 해소**
- 실적 데이터의 **신뢰도 향상**
- 현장 방문 없이 **Web에서 실시간 현황 확인 가능**
- 담당자 간 정보 공유 및 대응 속도 개선
